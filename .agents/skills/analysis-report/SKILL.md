# ANALYSIS REPORT

## 技能目标
你的核心任务是检查当前工作区中已经运行产生的**结果文件**和**日志**，判断它们是否与指定的计划文件（如 `plan/PLAN_0.md` 或其他 `PLAN_xx.md`）中的要求完全吻合。

## 严格限制
1. **禁止检查代码**：绝对不要读取、分析或审查任何源代码文件。你的注意力必须 100% 集中在“执行结果”、“输出文件”和“运行日志”上。
2. **作用域限制**：仅在 `stnet_syllabus` 目录下进行相关文件的检查与比对。

## 执行步骤

当被唤醒执行此技能时，请严格按照以下步骤操作：

### Step 1: 明确目标 Plan
- 如果用户没有明确指出需要检查哪个 Plan，请先询问用户（例如：“请问这次需要与哪一个 `PLAN_xx.md` 进行比对？”）。
- 读取对应的目标 `PLAN_xx.md`，提取出核心需求、预期输出格式以及验收标准。

### Step 2: 收集结果与日志
- 浏览 `workspace/stnet_syllabus` 目录，寻找与该 Plan 相关的实际运行结果（输出文件、数据等）。
- 仔细阅读运行期间产生的日志文件（Logs）。

### Step 3: 一致性比对
- **比对结果**：实际输出的内容、格式和数据是否满足 Plan 中定义的所有要求？是否存在遗漏？
- **排查日志**：日志中是否存在 Error、Exception、Warning 或与 Plan 预期不符的异常行为？

### Step 4: 汇报与输出
- **发现问题**：如果检查出任何不一致的地方、未完成的要求或异常日志，你必须将完整的分析汇报写入到工作区内的 `ANALYSIS_REPORT` 文件中。
  - `ANALYSIS_REPORT` 必须结构清晰，且包含以下内容：
    1. **核对目标**：当前检查的 Plan 文件名（例如：`PLAN_0.md`）。
    2. **异常概述**：一句话总结发现的核心问题。
    3. **详细不一致列表**：逐条列出 Plan 要求与实际结果的差异。
    4. **日志分析**：附上相关的错误或异常日志片段及其解析。
- **无问题**：如果结果完美符合且日志无异常，请直接在对话中告知用户“检查通过，运行结果与计划要求完全一致”，**无需**生成 `ANALYSIS_REPORT`（除非用户明确要求生成）。
